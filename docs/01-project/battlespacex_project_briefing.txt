
WARZONE - BATTLESPACEX | PROJECT BRIEFING FILE
Purpose: Quick briefing to explain the project to another AI agent, developer, or collaborator.

------------------------------------------------------------
1. PROJECT SUMMARY
------------------------------------------------------------

BattlespaceX is a military‑style global conflict monitoring and OSINT visualization platform.

The system renders real‑time or near‑real‑time geopolitical and military‑related activity on a 3D globe using CesiumJS.

The platform aggregates multiple open‑source signals and visualizes them with cinematic command‑center style UI components such as missile arcs, hotspots, alerts, and military movement tracks.

The goal is to build a high‑performance strategic visualization interface similar to platforms like strikemap.live but with more advanced intelligence‑style visualization layers.

Key capabilities include:

• Missile strike visualization
• Drone / UAV activity
• Military aircraft tracking
• Naval activity tracking
• Airspace alerts
• Cyber signals
• Thermal / anomaly events
• OSINT event aggregation
• Real‑time / near‑real‑time updates

The interface is designed to resemble a military operations dashboard rather than a traditional map.

------------------------------------------------------------
3. KEY FRONTEND MODULES
------------------------------------------------------------

warzone-globe.js

Responsible for:
• Cesium viewer initialization
• globe rendering
• map layers
• terrain and satellite modes
• rendering event markers
• rendering missile arcs
• updating hotspot anchors
• reporting viewport bounds

essential.js

Main application controller.

Handles:

• application initialization
• event normalization
• region filtering
• layer visibility
• syncing events to the globe
• hotspot updates
• military track updates
• polling live updates
• viewport-based data loading

warzone-military-tracks.js

Responsible for:

• aircraft tracks
• drone tracks
• naval tracks
• trail visualization
• animated movement paths

warzone-layers.js

Responsible for:

• layer definitions
• layer toggles
• event category visibility

warzone-realtime.js

Responsible for:

• polling feeds
• ingesting live events
• updating the platform dataset

6. VISUAL DESIGN GOALS
------------------------------------------------------------

The platform should look like a military command center.

Key design characteristics:

• cinematic glow effects
• radar style colors
• missile arc animations
• aircraft track animations
• siren alerts
• minimal UI clutter

------------------------------------------------------------
7. PERFORMANCE STRATEGY
------------------------------------------------------------

To maintain performance:

• viewport-based data loading
• event clustering
• hotspot pooling
• debounced rendering
• efficient Cesium entity management

The system should remain smooth even with large numbers of events.

------------------------------------------------------------
8. CURRENT KNOWN ISSUES
------------------------------------------------------------

• hotspot cards sometimes flicker
• performance degradation after some changes
• map label quality inconsistencies
• some features temporarily missing after optimization attempts

The system relies heavily on the coordination between:

essential.js
warzone-globe.js

Major rewrites of those files can break functionality.
