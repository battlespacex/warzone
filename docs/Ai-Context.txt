Project Summary
---------------
BattlespaceX is a military‑style global conflict monitoring and OSINT visualization platform. 
It displays real‑time or near‑real‑time geopolitical and military‑related activity on a 3D globe using CesiumJS.

The platform aggregates open‑source signals and visualizes them through cinematic command‑center style UI elements such as:

• missile strike arcs
• drone/UAV activity
• military aircraft tracks
• naval activity
• airspace alerts
• cyber indicators
• thermal or anomaly signals
• escalation alerts
• event hotspots

The interface is designed to resemble a **military operations theater dashboard**, not a traditional web map.

Core Frontend Modules
---------------------

essential.js
Main application controller.

Handles:
• application initialization
• event normalization
• region filtering
• layer control
• hotspot updates
• military track updates
• syncing event data to the globe
• live polling

Responsible for:
• globe initialization
• map rendering
• satellite/terrain layers
• event markers
• missile arcs

• viewport bounds reporting

Responsible for:
• aircraft tracks

warzone-layers.js
Responsible for:
• layer definitions
• layer toggles
• category visibility

warzone-realtime.js
Responsible for:
• polling feeds

Performance Strategy
--------------------

To keep the globe fast:

• viewport based event loading
• event clustering
• hotspot pooling
• debounced updates
• efficient Cesium entity usage

When the map moves:

1. camera stops
2. visible bounds are calculated
3. events for that region are fetched
4. only visible events are rendered

Known Issues (Current Development Phase)
----------------------------------------

• hotspot cards sometimes flicker
• performance degradation after some rendering changes
• map label quality inconsistencies
• some visual features temporarily missing after optimization attempts

The two most sensitive files in the system are:

essential.js
warzone-globe.js

Large rewrites in these files can break the platform.
